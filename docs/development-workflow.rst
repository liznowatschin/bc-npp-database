Development Workflow
====================

BC-NPPD follows the UBC-FRESH phase/task/subtask workflow.

* `ROADMAP.md` is the current plan and issue tracker map.
* One roadmap phase maps to one GitHub parent issue and one feature branch.
* One roadmap task maps to one child issue linked from the parent issue body.
* Subtasks usually stay as checklist items inside the child issue body.
* `CHANGE_LOG.md`, roadmap entries, planning notes, issue comments, and pull
  request descriptions should stay synchronized.

Public repository content must stay clean of private data, raw transcripts,
unpublished source material, generated local outputs, credentials, and
machine-specific paths.

GitHub templates
----------------

Use the parent phase issue template for roadmap phases, the child task issue
template for roadmap tasks, and the pull request template for phase closeout.
Issue bodies and pull requests should be readable as project specifications, not
just status notes.
