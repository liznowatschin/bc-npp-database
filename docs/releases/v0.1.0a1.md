# v0.1.0a1

BC-NPPD `v0.1.0a1` is the first GitHub alpha prerelease for the Vancouver
plant list proof-of-concept product.

## Highlights

- Adds a tracked 20-species Vancouver/CDF PoC plant list with stable
  `BCNPPD-*` IDs.
- Adds source registry and source-attribution sidecars for the PoC records.
- Adds evidence hardening artifacts that distinguish PoC-reviewed
  identity/native-range fields from candidate use and horticultural fields.
- Adds a static usability interface at `data/poc/vancouver/usability/index.html`.
- Adds search, candidate view filters, and click-to-open plant record detail
  panels with attributes, sources, attribution, evidence gaps, score readiness,
  use-case memberships, and caveats.

## Candidate Views

The static interface includes candidate filters for boulevard, rain garden, dry
sun, and shade. It also includes a pollinator review queue and a low-growing
view marked `insufficient_data`.

## Caveats

- This is a proof-of-concept product, not final planting advice.
- Use-case views are candidate inspection aids.
- UNI, PSI, and RVI scores remain `not_ready`.
- Horticultural and score-related fields still need field-specific reviewed
  evidence before promotion.
- No PyPI release is included in `v0.1.0a1`; this is a GitHub prerelease with
  wheel and source distribution artifacts attached.
