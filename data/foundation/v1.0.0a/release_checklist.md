# v1.0.0a Foundation Release Checklist

This checklist prepares the alpha foundation release. It does not create a tag
or publish a GitHub release by itself.

- [x] Governance, roadmap, changelog, and source policy are in place.
- [x] Source-attribution records and validators exist.
- [x] Canonical import/export records and validators exist.
- [x] Provisional scoring records and validators exist.
- [x] One public-safe `Achillea millefolium` foundation record is tracked.
- [x] Raw PDFs, screenshots, generated outputs, private data, and local scratch
      files remain ignored.
- [ ] Before tagging, rerun full acceptance on `main`.
- [ ] Before tagging, confirm GitHub CI and Docs workflows pass.
- [ ] Before tagging, confirm maintainers approve release notes and artifact
      contents.
- [ ] Create tag `v1.0.0a` only after approval.
